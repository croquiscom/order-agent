# /check-logs - 최근 실행 결과 분석

## 목적
시나리오 실행 후 logs/ 디렉터리에 쌓인 로그와 스크린샷을 빠르게 분석한다.
400+ 파일 속에서 최근 결과만 골라 성공/실패를 즉시 파악할 수 있다.

## 사용 시점
- 시나리오 실행 직후 결과 확인
- 특정 시간대의 실행 결과 조회
- FAILED 라인만 빠르게 추출할 때

## 실행 방법

1. `logs/` 디렉터리에서 가장 최근 `order-agent-exec_*.log` 파일을 찾는다.
   - $ARGUMENTS에 날짜/시간 힌트가 있으면 해당 시간대 로그를 탐색한다.
   - $ARGUMENTS에 파일명이 있으면 해당 파일을 직접 분석한다.

2. 로그 파일에서 아래 패턴을 추출한다:
   - `FAILED` — 실패한 액션과 라인 번호
   - `ERROR` — 에러 메시지
   - `OK` — 성공한 액션 수
   - 실행 시작/종료 시간, 총 소요 시간

3. 동일 타임스탬프의 관련 파일을 함께 확인한다:
   - `failed_line_*.png` — 실패 시점 스크린샷
   - `diag_*.png` / `diag_*.txt` — 진단 캡처
   - `result.png` — 최종 화면

4. 결과 요약을 제공한다:
   - 전체 커맨드 수 / 성공 / 실패
   - 실패 항목별 라인 번호, 액션, 에러 메시지
   - 관련 스크린샷 목록

## 예시
- `/check-logs` — 가장 최근 로그 분석
- `/check-logs 20260307` — 3/7 로그만 분석
- `/check-logs order-agent-exec_20260306_214920.log` — 특정 로그 분석
