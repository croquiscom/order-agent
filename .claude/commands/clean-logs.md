# /clean-logs - 오래된 로그 정리

## 목적
logs/ 디렉터리에 무한 축적되는 로그 파일과 스크린샷을 정리한다.
이틀 분량의 테스트만으로도 400+ 파일이 쌓이므로 주기적 정리가 필요하다.

## 사용 시점
- 디스크 공간 확보가 필요할 때
- 오래된 테스트 결과를 정리할 때
- 새 테스트 세션을 깨끗하게 시작할 때

## 실행 방법

1. $ARGUMENTS에서 보존 기간을 파싱한다 (기본: 3d = 3일)
   - 숫자+d: 일 단위 (예: 7d = 7일)
   - 숫자+h: 시간 단위 (예: 24h = 24시간)
   - `all`: 전체 삭제 (.gitkeep 제외)

2. `logs/` 디렉터리에서 보존 기간 이전 파일을 수집한다:
   - `*.log` — 실행 로그
   - `*.png` — 스크린샷
   - `*.txt` — 텍스트 진단
   - `*.json` — JSON 덤프
   - `.gitkeep`는 항상 제외

3. 삭제 대상을 사용자에게 보여준다:
   - 파일 수
   - 총 용량
   - 가장 오래된/최근 파일 날짜

4. **반드시 사용자 확인을 받은 후** 삭제를 실행한다.

5. 정리 결과를 보고한다:
   - 삭제된 파일 수 / 확보된 용량
   - 남은 파일 수

## 예시
- `/clean-logs` — 3일 이전 로그 정리
- `/clean-logs 7d` — 7일 이전 로그 정리
- `/clean-logs 12h` — 12시간 이전 로그 정리
- `/clean-logs all` — 전체 정리 (.gitkeep 제외)
